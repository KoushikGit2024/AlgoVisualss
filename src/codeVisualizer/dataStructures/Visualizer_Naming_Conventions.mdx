# AlgoVisuals Naming Conventions

> The `VisualGround` engine's heuristic detector parses the AST runtime scope and automatically mounts the correct React component based on variable naming prefixes **and value shape**. Follow the conventions below to trigger the right visual for each data structure.

---

## Detection Philosophy

The detector uses a **two-phase** approach:

1. **PREFIX match** — Does the variable name start with a known trigger prefix?
2. **SHAPE validation** — Is the runtime value actually the correct data shape?

This means `numRows = 5` will **not** trigger `<D1Array />` even though it starts with `num`, because `5` is a scalar, not an array. Only variables whose value passes both checks get visualized.

### Priority Order

When multiple variables match different detectors, the **first match wins** in this order:

| Priority | Component | Why first? |
|----------|-----------|------------|
| 1 | `<Graph />` | Most specific prefix (`adj_`, `graph_`) |
| 2 | `<Tree />` | Requires `tree_`/`bst_` + node objects |
| 3 | `<D2Array />` | 2D array shape (matrix) |
| 4 | `<LinkedList />` | Requires `ll_`/`head` + node objects |
| 5 | `<Stack />` | Flat array with LIFO semantics |
| 6 | `<Queue />` | Flat array with FIFO semantics |
| 7 | `<D1Array />` | Generic flat array fallback |

## 1. Graphs — `<Graph />`

Renders a circular node-edge network.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `adj` | `vector<vector<int>> adj_list` |
| `graph` | `unordered_map<int, vector<int>> graph_map` |

> **Shape**: Value must be a 2D array (adjacency list) or a non-empty array.

**Auxiliary Variables (Auto-Bound)**

| Role | Trigger | Notes |
|------|---------|-------|
| Edge list | Any variable containing `edge` (e.g., `graph_edges`, `edges`) | Expected format: `[[u, v], [x, y]]` |
| Visited state | Any variable containing `visit` (e.g., `visited`, `node_visits`) | Elements set to `1` pulse with the highlight color |
| Pointer badges | `node`, `curr`, `u`, `v`, or `ptr_n*` prefix | **All** matching variables render as floating badges simultaneously |

**Example — BFS Graph Walk**

```cpp
void bfs() {
    vector<vector<int>> adj_list;   // Triggers <Graph />
    vector<vector<int>> graph_edges; // Auto-bound: edge list
    vector<int> visited;             // Auto-bound: highlight visited nodes
    int curr_node = 0;               // Auto-bound: floating pointer badge
    int u = 0;                       // Also auto-bound: second badge
}
```

## 2. Trees & BSTs — `<Tree />`

Renders a top-down hierarchical tree. Node coordinates are auto-calculated using an in-order layout algorithm.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `tree_` | `vector<TreeNode> tree_nodes` |
| `bst_` | `vector<Node> bst_data` |
| `trie_` | `vector<TrieNode> trie_nodes` |

> **Shape**: Value must be an array of objects, each with `id` and `value` fields.

**Required Node Data Structure**

```json
[
  { "id": "0", "value": "10", "left": "1", "right": "2" },
  { "id": "1", "value": "5" },
  { "id": "2", "value": "15" }
]
```

**Auxiliary Variables (Auto-Bound)**

| Role | Trigger |
|------|---------|
| Core pointers | `root`, `curr`, `parent`, `temp` |
| Branch pointers | `left`, `right` |

> Variables must store the target node's `id` (string or number). Badges attach above the target node.

---

## 3. 2D Matrices — `<D2Array />`

Renders a 2D grid/matrix layout.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `mat` | `vector<vector<int>> mat_grid` |
| `grid` | `vector<vector<int>> grid` |
| `board` | `char board[8][8]` |
| `dp` | `vector<vector<int>> dp_table` |

> **Shape**: Value must be a 2D array (array of arrays).

**Auxiliary Variables (Auto-Bound)**

Both a row **and** a column variable must exist in the same scope to drop a pointer.

| Role | Trigger |
|------|---------|
| Row | `r`, `row`, `*_r` (e.g., `ptr_r`), `r_*` (e.g., `r_curr`) |
| Column | `c`, `col`, `*_c` (e.g., `ptr_c`), `c_*` (e.g., `c_curr`) |

---

## 4. 1D Arrays — `<D1Array />`

Renders a linear array.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `arr` | `vector<int> arr` |
| `vec` | `vector<string> vec_names` |
| `nums` | `vector<int> nums` |
| `seq` | `vector<float> seq` |

> **Shape**: Value must be a flat array of primitives (numbers, strings, booleans).
>
> **Note**: `num` (singular) was removed to prevent false matches on `numRows`, `numCols`, etc.

**Auxiliary Variables (Auto-Bound)**

| Role | Trigger |
|------|---------|
| Generic indexers | `i`, `j`, `k` |
| Algorithmic pointers | `left`, `right`, `mid`, `curr`, `ptr` |
| Composite examples | `ptr_i`, `left_idx` |

---

## 5. Linked Lists — `<LinkedList />`

Renders a horizontal sequence of connected node blocks.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `ll_` | `vector<Node> ll_nodes` |
| `head` | Engine emits the fully traversed list as `head` |

> **Shape**: Value must be an array of objects, each with `id` and `value` fields.

**Auxiliary Variables (Auto-Bound)**

| Role | Trigger |
|------|---------|
| Standard pointers | `head`, `tail`, `curr`, `prev`, `next` |
| Fast/Slow pointers | `slow`, `fast` |

> Variables must hold the `id` or index of the target node.

**Example — Cycle Detection**

```cpp
void detectCycle() {
    vector<Node> ll_nodes;       // Triggers <LinkedList />
    vector<int> highLightNodes;  // Optional: visual pulse on match
    vector<int> readNodes;       // Optional: mark read nodes

    Node* slow = head;           // Auto-bound: floating pointer
    Node* fast = head;           // Auto-bound: floating pointer

    while (fast && fast->next) {
        if (slow == fast) {
            highLightNodes.push_back(slow->id);
        }
    }
}
```

## 6. Queues & Deques — `<Queue />`

Renders a horizontal FIFO tube where items slide in from the right and exit from the left.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `q_` | `queue<int> q_nodes` |
| `queue` | `deque<string> queue_names` |
| `deque` | `deque<int> deque_window` |

> **Shape**: Value must be a flat array of primitives.

**Auxiliary Variables (Auto-Bound)**

| Role | Trigger |
|------|---------|
| Standard ends | `front`, `back`, `rear`, `head`, `tail` |
| General trackers | `curr`, `ptr_front` |

---

## 7. Stacks — `<Stack />`

Renders a vertical LIFO bucket where elements animate strictly top-down.

**Primary Structure Prefixes**

| Prefix | Example |
|--------|---------|
| `st_` | `stack<int> st_nodes` |
| `stack` | `vector<string> stack_names` |

> **Shape**: Value must be a flat array of primitives.

**Auxiliary Variables (Auto-Bound)**

| Role | Trigger |
|------|---------|
| Stack pointers | `top`, `peek` |

> Pointer badges attach to the **right side** of the corresponding element.

---

## Reserved Variables

These variable names are **never consumed** by the detector and will always appear in the Variables panel:

| Variable | Why reserved |
|----------|-------------|
| `n` | Common loop counter / size variable |
| `maxVal`, `minVal` | Computed results |
| `result`, `ans` | Return values |
| `temp` (when scalar) | Only consumed by Tree detector when holding a node ID |

## Quick Reference

| Component | Primary Prefixes | Shape Required | Key Auxiliary Variables |
|-----------|-----------------|----------------|------------------------|
| `<Graph />` | `adj`, `graph` | 2D array | `*edge*`, `*visit*`, `node`/`curr`/`u`/`v` (all matched) |
| `<Tree />` | `tree_`, `bst_`, `trie_` | Array of `{id, value}` | `root`, `curr`, `parent`, `temp`, `left`, `right` |
| `<D2Array />` | `mat`, `grid`, `board`, `dp` | 2D array | `r`/`row`/`r_*`, `c`/`col`/`c_*` (both required) |
| `<D1Array />` | `arr`, `vec`, `nums`, `seq` | Flat array | `i`, `j`, `k`, `left`, `right`, `mid`, `curr`, `ptr` |
| `<LinkedList />` | `ll_`, `head` | Array of `{id, value}` | `head`, `tail`, `curr`, `prev`, `next`, `slow`, `fast` |
| `<Queue />` | `q_`, `queue`, `deque` | Flat array | `front`, `back`, `rear`, `head`, `tail`, `curr` |
| `<Stack />` | `st_`, `stack` | Flat array | `top`, `peek` |
